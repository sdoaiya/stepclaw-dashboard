# 灵笼看板 v2.1 优化需求文档 (PRD)

**文档版本**: v1.0  
**创建日期**: 2026-03-28  
**创建人**: 灵析 (需求分析师)  
**状态**: 评审中

---

## 1. 项目背景

### 1.1 问题陈述
当前 v2.0 看板存在以下核心问题：
- **视觉降级**: 页面设计比原版 v4.x 差很多，缺乏精致感
- **功能缺失**: 核心功能被删减，用户体验下降
- **交互简化**: 缺少动态效果和交互反馈

### 1.2 目标
在保持 GitHub Pages 静态托管的前提下，恢复并超越原版 v4.x 的视觉体验和功能完整性。

---

## 2. 现状分析

### 2.1 当前 v2.0 问题清单

| 问题类别 | 具体问题 | 严重程度 |
|---------|---------|---------|
| **布局** | 单栏简单布局，缺少层次感 | 🔴 高 |
| **视觉** | 静态卡片，无玻璃态效果 | 🔴 高 |
| **功能** | 无部门筛选功能 | 🔴 高 |
| **交互** | 无点击高亮、悬停效果弱 | 🟡 中 |
| **数据** | 16员工展示，但信息密度低 | 🟡 中 |
| **动效** | 缺少过渡动画 | 🟡 中 |

### 2.2 原版 v4.x 核心优势

1. **三栏沉浸式设计**: 左统计/部门、中员工、右任务/动态
2. **玻璃态美学**: backdrop-filter + 渐变 + 动态光效
3. **部门筛选**: 点击部门高亮对应员工
4. **实时状态**: 15员工完整信息 + 实时更新
5. **动态 Feed**: 任务列表 + 活动流
6. **响应式适配**: PC三栏/移动单栏无缝切换

## 4. 功能优先级

### 4.1 优先级矩阵

```
        高影响
           │
    P0     │    玻璃态效果
  必须做   │    三栏布局
           │    部门筛选
           │    实时统计
           │
───────────┼───────────────
           │
    P1     │    动态Feed
  应该做   │    主题切换
           │    部门分布统计
           │
           │
    P2     │    高级动画
  可以做   │    自定义配置
           │
           └───────────────
              低复杂度    高复杂度
```

### 4.2 优先级列表

#### P0 - 核心功能 (必须实现)
1. **玻璃态卡片效果** - 所有面板使用 backdrop-filter 实现毛玻璃
2. **三栏响应式布局** - PC三栏/移动单栏自适应
3. **部门筛选高亮** - 点击部门筛选对应员工
4. **实时统计面板** - 4个核心指标卡片
5. **15员工完整展示** - 头像、状态、任务信息
6. **任务列表** - 优先级标识、负责人、时间
7. **固定头部导航** - 滚动时保持可见
8. **状态呼吸灯动画** - 在线状态动态效果

#### P1 - 重要功能 (应该实现)
1. **动态 Feed 流** - 实时活动展示
2. **数据刷新动画** - Loading 过渡效果
3. **部门分布统计** - 各部门人数统计
4. **主题切换** - 明暗模式切换

#### P2 - 增强功能 (可选实现)
1. **高级交互动画** - 页面切换、数据更新动画
2. **自定义刷新频率** - 用户可配置刷新间隔
3. **键盘快捷键** - 快速切换视图

## 5. 技术约束

### 5.1 技术栈限制

| 约束项 | 要求 | 说明 |
|-------|------|------|
| 托管平台 | GitHub Pages | 纯静态文件托管 |
| 前端技术 | HTML5 + CSS3 + Vanilla JS | 无框架依赖 |
| 数据格式 | JSON | linglong-data.json |
| 更新机制 | GitHub Actions | 每2分钟自动更新 |
| 浏览器兼容 | Chrome/Firefox/Safari/Edge | 最新2个版本 |

### 5.2 性能约束

- **文件大小**: 单个 HTML < 100KB, CSS/JS < 50KB
- **图片资源**: 使用 emoji 或 SVG，无外部图片依赖
- **CSS 动画**: 优先使用 transform/opacity，避免触发重排
- **数据请求**: 使用 fetch API，支持缓存控制

### 5.3 设计约束

- **颜色系统**: 使用 CSS 变量，支持主题切换
- **字体**: 系统默认字体栈，无外部字体加载
- **响应式断点**: 768px (平板/手机分界)
