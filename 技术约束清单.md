## 1. 技术栈约束

### 2.3 特性检测代码

```javascript
// 玻璃态效果特性检测
const supportsBackdropFilter = CSS.supports('backdrop-filter', 'blur(10px)');

// 应用降级样式
if (!supportsBackdropFilter) {
  document.body.classList.add('no-backdrop-filter');
}
```


## 3. 性能约束

### 3.1 文件大小限制

| 文件类型 | 最大大小 | 说明 |
|---------|---------|------|
| HTML | 100KB | 包含内联 CSS/JS |
| CSS (外链) | 50KB | 如使用外链样式 |
| JS (外链) | 50KB | 如使用外链脚本 |
| JSON 数据 | 100KB | 数据文件大小 |
| 图片资源 | 0KB | 禁止使用外部图片 |

### 3.2 性能指标

| 指标 | 目标值 | 测量工具 |
|-----|-------|---------|
| First Contentful Paint (FCP) | < 1.5s | Lighthouse |
| Largest Contentful Paint (LCP) | < 2.5s | Lighthouse |
| Time to Interactive (TTI) | < 3.5s | Lighthouse |
| Cumulative Layout Shift (CLS) | < 0.1 | Lighthouse |
| 动画帧率 | 60fps | Chrome DevTools |

### 3.3 性能优化要求

1. **CSS 优化**
   - 使用 `transform` 和 `opacity` 做动画
   - 避免触发重排的属性 (width, height, top, left)
   - 使用 `will-change` 谨慎优化动画元素

2. **JavaScript 优化**
   - 使用 `requestAnimationFrame` 做动画
   - 防抖/节流高频事件 (resize, scroll)
   - 及时清理定时器和事件监听

3. **数据加载优化**
   - 使用 `fetch` 的缓存控制
   - 数据更新时局部刷新 DOM
   - 避免频繁的全量重绘

### 6.1 数据结构

```typescript
// linglong-data.json 结构
interface DashboardData {
  status: 'ok' | 'error';
  timestamp: string; // ISO 8601 格式
  version: string;

  title: string;
  priority: 'high' | 'medium' | 'low';

interface Activity {
  id: string;
  type: 'task_start' | 'task_complete' | 'status_change';

}
```

## 7. 代码规范

### 7.1 HTML 规范

- 使用语义化标签 (`header`, `main`, `section`, `article`)
- 所有图片使用 `alt` 属性 (即使是装饰性 emoji)
- 表单元素使用 `label` 关联
- 语言属性 `<html lang="zh-CN">`

### 7.2 CSS 规范

- 使用 BEM 命名规范
- CSS 变量定义在 `:root`
- 媒体查询放在文件底部
- 避免使用 `!important`

### 7.3 JavaScript 规范

- 使用 `const` 和 `let`，避免 `var`
- 使用箭头函数
- 使用模板字符串
- 使用解构赋值
- 添加必要的注释

### 7.4 代码组织

```
linglong.html
├── <style>
│   ├── CSS Variables
│   ├── Reset & Base
│   ├── Layout
│   ├── Components
│   ├── Animations
│   └── Media Queries
└── <script>
    ├── Constants
    ├── State
    ├── DOM Elements
    ├── Functions
    │   ├── init()
│   ├── loadData()
│   ├── renderXxx()
│   └── event handlers
```

## 9. 部署约束

### 9.1 GitHub Actions 工作流

```yaml
name: Update Dashboard Data
on:
  schedule:
    - cron: '*/2 * * * *'  # 每2分钟
  workflow_dispatch:

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Generate Data
        run: node generate-data.js
      - name: Commit and Push
        run: |
          git config user.name "GitHub Action"
          git config user.email "action@github.com"
          git add linglong-data.json
          git commit -m "Update data: $(date)" || exit 0
          git push
```
